# 项目定位说明：live-auction-user-h5

## 一句话定位

`live-auction-user-h5` 是直播竞拍系统的**用户/观众移动 H5**，负责让买家进入直播间、观看直播、参与出价、查看结果并模拟支付。

## 主体是谁

本项目面向的是**普通观众/买家**：

- 进入直播间的潜在买家。
- 正在参与某件拍品竞拍的出价用户。
- 竞拍结束后查看成交结果、订单状态和模拟支付的用户。

## 本项目负责什么

- 移动端直播间体验：直播区域、当前拍品、价格、倒计时、排名。
- 出价交互：登录/注册、出价输入、固定加价提示、出价结果反馈。
- 实时提醒：领先、被超越、自动延时、竞拍结束、封顶价成交等状态提示。
- 断线恢复：WebSocket 心跳、重连、快照恢复后的页面状态修正。
- 成交闭环：查看竞拍结果、订单信息、模拟支付入口。
- 移动端适配：H5 竖屏优先，手指操作、弱网提示、直播间氛围感。

## 本项目不负责什么

- 不做主播团队 PC 工作台，不提供添加拍品、排品、控场、复盘等后台能力。
- 不做后端业务状态机、出价一致性或订单生成逻辑；这些由 `live-auction-bid-backend` 保证。
- 不使用商品图假装真实直播；直播模拟必须通过播放器/视频流封装表达。
- 生产模式不允许静默创建 demo 买家账号。

## 和其它项目的关系

- 调用 `live-auction-bid-backend` 的房间快照、出价、结果、订单支付和 WebSocket 契约。
- 与 `live-auction-bid-frontend` 共同围绕同一个直播间协作：PC 端负责主播团队控场，H5 负责观众竞拍。
- H5 是课题评分中的移动端观众体验入口，不能被 PC 后台页面替代。

## 文档维护要求

所有移动端用户路径、弱网/重连策略、出价交互、结果页和支付流程变化都必须写入 `docs/`，确保 H5 始终作为独立主体维护。
