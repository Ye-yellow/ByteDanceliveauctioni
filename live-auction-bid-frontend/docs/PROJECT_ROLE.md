# 项目定位说明：live-auction-bid-frontend

## 一句话定位

`live-auction-bid-frontend` 是直播竞拍系统的**主播团队 PC 工作台**，也可以叫 LiveAuction Studio / 直播间竞拍工作台。

## 主体是谁

本项目面向的是**主播团队内部人员**：

- 主播/商家主账号：由平台或公司后台发放，绑定当前商家空间和固定直播间。
- 主播：查看今日拍品、开拍、落锤、异常取消。
- 场控：直播中观察价格、倒计时、排名、WebSocket 同步状态。
- 商品助理：添加拍品、配置竞拍玩法、准备讲解卡、加入今日队列。
- 订单客服：处理成交、支付、履约相关工作。
- 数据复盘人员：直播后查看拍品表现和复拍线索。

## 本项目负责什么

- 直播前：添加拍品/创建竞拍拍品、配置竞拍规则、本场拍品队列/今日排品。
- 直播中：直播间中控台、当前价/倒计时/排名展示、自动延时提示、异常取消、落锤成交、切换下一件。
- 直播后：成交处理、支付履约入口、拍品表现复盘、复制重拍。
- 后台状态：API 连接状态、WebSocket 实时链路诊断、错误提示和空状态。
- 前端工程：React + TypeScript + Vite 页面结构、真实后端契约接入、前端类型生成。

## 本项目不负责什么

- 不做用户 H5、观众直播间入口、小程序页面；这些属于 `live-auction-user-h5` 或未来小程序项目。
- 不做买家账号注册、创建或管理；买家身份只在 H5/用户端产生和使用。
- 不做主播/商家主账号注册或创建；主账号由平台或公司后台发放，当前前端只允许主账号管理其团队子账号。
- 不做平台级多直播间管理，不使用“平台管理员后台”口吻。
- 不新增 mock 服务或 mock 数据；缺少后端契约的能力只能禁用并记录契约需求。
- Home 页视觉已冻结，除非明确要求解冻，不主动修改主视觉、随机元素布局、主卡比例和按钮样式。

## 和其它项目的关系

- 调用 `live-auction-bid-backend` 的 HTTP API 和 WebSocket。
- 与 `live-auction-user-h5` 共享同一竞拍业务事实：同一个房间、同一个拍品、同一个价格/排名/倒计时。
- 本项目展示“主播团队看到什么”，H5 展示“买家/观众看到什么”。

## 文档维护要求

所有 PC 工作台的信息架构、页面命名、设计规范、接口契约变化都要进入 `docs/`，避免后续又混成平台后台或用户端。
