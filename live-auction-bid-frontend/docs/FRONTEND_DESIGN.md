# Frontend Design

React 直播竞拍控制台/观众端初版，已对齐后端 JWT 鉴权和统一响应包装。

## 页面能力

- Home 产品落地页：根据 PDF 课题背景展示直播竞拍闭环、评分硬点和商家主播后台入口；
- Login 独立登录页：主播/运营与观众买家双模式，登录后再进入后台或直播间；
- 直播竞拍房间状态展示；
- WebSocket 连接状态；
- 当前拍品价格、最低下一口、倒计时；
- 实时排行榜；
- 买家注册/登录后出价；
- 主播/运营/admin 登录后创建、开拍、揭示信任卡、Duel、落锤、异常取消；
- 业务错误展示来自 `reply.result.message`。

## 后端连接

默认同源代理，部署时可覆盖：

```text
VITE_API_BASE=
VITE_WS_BASE=ws://当前前端 host
```

真实后端默认端口：`18080`。

## 当前前端项目设计边界

当前 `live-auction-bid-frontend` 是 **商家主播/运营后台 Web**，不是“后台 + 用户 H5”的双入口项目。

- **当前项目只做后台**：商品上架、规则配置、开拍/取消/结束、订单管理、数据看板，PC 管理后台优先。
- **用户 H5 竞拍端另起项目**：移动端优先，承担用户实时出价、排名/倒计时同步、提醒、订单结果与模拟支付。
- **小程序端也是独立端**：若后续需要，按微信/抖音小程序规范单独开发，复用后端 API/WebSocket。

因此 Home 页不再放“用户 H5 竞拍端 + 商家主播后台”两个同级入口，而只保留进入商家主播后台的入口；H5/小程序作为系统架构中的独立端记录在文档中。

## Home / Login 视觉取舍

用户指出前端不能直接进入管理后台，应参考 Sub2API 的 `/home` 与 `/login`。本轮增加公开入口层：

- **Home 页**：采用浅色网格背景、轻量品牌区、Hero 文案、系统流程、能力卡片和评分点 chips；内容改成“实时竞拍大师”，贴合《抖音电商AI全栈课题-直播竞拍全栈系统》背景。
- **Login 页**：采用居中品牌 + 登录卡片，默认主播/运营账号，支持切换观众买家注册/登录；登录成功后通过 `?next=/host` 或角色默认路径跳转。
- **架构分层**：`/home` 负责说明当前后台产品，`/login` 负责身份，`/host`/`/admin` 负责商家主播后台。用户 H5/小程序端作为后续独立项目，不在当前前端中提供同级入口。
- **不伪造业务数据**：Home 只展示课题和系统能力；Login 只调用真实 `login/register`；后台和观众端继续使用真实 API/WebSocket。

## 主播后台视觉取舍

主播后台按 **PC 管理后台 + 直播控场台** 设计，而不是移动观众端的沉浸式直播间：

- **顶部 hero**：明确当前是 Host Control Room，并提供观众端入口与刷新后端状态按钮，方便答辩/联调时在两个角色间切换。
- **状态总览**：把拍品总数、直播中、草稿、成交/取消、信任揭示进度前置，满足主播控场时“先看系统状态，再操作”的管理后台信息架构。
- **当前拍品为主工作台**：当前价、状态、版本号、结束时间、领先用户、阶段、防狙击规则集中展示；开拍/进入 Duel/落锤是主操作与次操作分层，按钮启用条件直接反映后端状态机。
- **创建拍品在左侧 rail**：作为后台常驻工具，不抢当前直播工作台的视觉优先级；仍然只调用真实 `CreateLot`，不内置 mock。
- **信任揭示独立分区**：卡片揭示是直播成交信任链路的一部分，和价格控制区拆开，避免主播误操作。
- **危险操作分区**：异常取消放入 Danger Zone，使用原因输入 + 红色 danger 按钮，并仅对 `LOT_STATUS_LIVE` 启用；当前已接入真实 `POST /api/lots/{lot_id}/cancel` 契约，取消原因随请求提交后端。
- **响应式保底**：大屏使用左侧 rail + 右侧控场台；中屏折叠为单列 hero/工作台；小屏所有按钮和卡片纵向排列，保障演示和移动查看不崩。

本轮没有引入大型 UI 库；只在 `src/app/styles.css` 增加 CSS，用原生 React/TypeScript 组合现有 feature API。

## 设计取舍

- 当前不引入全局状态库，认证状态用一个轻量 Auth Token Adapter 管理；等页面复杂度提升后再考虑 Zustand/Redux。
- 不引入 OpenAPI 生成文件，因为后端当前没有最新 OpenAPI artifact；避免 stale schema 误导。
- WebSocket 事件先规范化 enum，再更新 React state；这是对后端 JSON 表达差异的防腐层，不是业务 fallback。
- 主播后台保留真实后端接入；若后续某个 P0 契约尚未落地，前端只可预留禁用 UI 与文档说明，不得调用不存在接口或写 mock 数据。

## Development Rule

Do not add mock services or mock data into this frontend repository. The frontend should connect to the real backend API/WebSocket contract, or use externally managed test environments.
