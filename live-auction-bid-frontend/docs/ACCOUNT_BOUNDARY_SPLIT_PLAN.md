# 账号体系拆分标记

Status: `ACTIVE_PRE_LAUNCH`

本文件用于标记账号体系的产品边界和后续拆分方向。当前尚未上线，不做旧版兼容；上线前如需要兼容旧数据或旧接口，再单独补兼容策略。

## 为什么要拆开

直播竞拍系统里有三类账号生命周期，不能混在同一个前端或同一个管理入口里：

- 平台/公司侧账号发放：公司内部给主播/商家开通主账号，绑定商家空间和固定直播间。
- 主播团队工作台账号：主播/商家主账号进入 LiveAuction Studio 后，管理自己空间下的团队子账号。
- 买家账号：用户在 H5/小程序注册、登录、出价、查看订单和支付结果。

如果这三类账号混在一起，会导致 Admin 前端误创建买家、主播团队误认为能注册主账号、后端接口权限边界不清，后续联调和上线都会出问题。

## 当前项目边界

`live-auction-bid-frontend` 只负责主播团队 PC 工作台：

- 允许：主播/商家主账号登录。
- 允许：主播/商家主账号管理当前商家空间下的团队子账号。
- 允许：团队子账号进入对应权限的拍品、队列、中控、成交处理页面。
- 禁止：创建主播/商家主账号。
- 禁止：创建、筛选、展示或管理买家账号。
- 禁止：提供用户 H5 注册、登录、出价入口。

## 后端边界

`live-auction-bid-backend` 需要提供清晰分离的接口：

- 平台/公司后台接口：负责开通主播/商家主账号和商家空间。
- Studio Admin 接口：负责当前商家空间内的团队子账号、拍品、队列、中控、成交处理。
- H5 用户接口：负责买家注册、登录、出价、订单结果、支付模拟。

当前阶段要求 fail-fast：

- Admin 创建团队账号接口不能创建 `USER_ROLE_BUYER`。
- Admin 改角色接口不能把用户改成 `USER_ROLE_BUYER`。
- Admin 团队账号列表默认不能返回买家账号。
- 显式用 Admin 查询买家账号应返回契约错误。

## H5 边界

`live-auction-user-h5` 负责买家视角：

- 买家注册和登录。
- 实时出价。
- 排行榜、倒计时、竞拍结果展示。
- 订单结果与支付模拟。

H5 不负责主播/商家主账号发放，也不负责主播团队子账号管理。
