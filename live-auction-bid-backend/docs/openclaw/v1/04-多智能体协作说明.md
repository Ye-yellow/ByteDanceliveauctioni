# V1 多智能体协作说明

## 1. 为什么要给每个角色建工作间

多人项目里，如果所有 agent 都在同一个文档里写，很容易出现：

- 上下文混乱；
- 职责重叠；
- 结论互相覆盖；
- 不知道某个判断是谁做的；
- 后续 agent 接手成本高。

因此 V1 为每个职责角色创建独立 docs 空间：

```text
docs/openclaw/v1/agents/{role}/
```

每个角色目录都像一个小型 agent workspace。

## 2. 每个角色目录包含什么

```text
README.md              角色说明
SOUL.md                角色智能体的人设、边界和工作方式
memory/MEMORY.md       角色长期记忆和关键决策
TASKS.md               任务板
HANDOFF.md             交接记录
outputs/               阶段产出
```

## 3. 怎么用

### 真人使用

团队成员进入自己负责的目录，例如：

```text
docs/openclaw/v1/agents/backend-auction-core/
```

先看：

```text
README.md
SOUL.md
TASKS.md
```

然后把产出放到：

```text
outputs/
```

### agent 使用

给 agent 分配任务时，可以指定：

```text
你是 docs/openclaw/v1/agents/system-architect/ 下定义的系统架构负责人。
先阅读该目录 README.md、SOUL.md、memory/MEMORY.md、TASKS.md。
然后完成 TASKS.md 中的第一项产出。
```

agent 完成后，应更新：

```text
memory/MEMORY.md
TASKS.md
HANDOFF.md
outputs/
```

## 5. 协作规则

- 角色内产出先放 `outputs/`；
- 汇总成正式团队文档后，再放到 `docs/openclaw/v1/` 根目录；
- 重要上下文写 `memory/MEMORY.md`；
- 跨角色依赖写 `HANDOFF.md`；
- 不在一个角色目录里替另一个角色做最终决策；
- 文档默认中文。
