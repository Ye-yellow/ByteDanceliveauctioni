# 项目定位说明：live-auction-bid-backend

## 一句话定位

`live-auction-bid-backend` 是直播竞拍全栈系统的**后端主服务**，负责把“主播开拍、买家出价、实时同步、落锤成交、订单生成、模拟支付”做成可信的业务闭环。

## 主体是谁

本项目的直接使用主体不是普通观众，而是给以下端提供统一能力：

- **主播团队 PC 工作台**：主播、场控、商品助理、订单客服、数据复盘人员通过后台前端调用本服务。
- **用户移动 H5**：观众/买家通过 H5 调用本服务参与竞拍、查看结果、模拟支付。
- **系统运维/答辩展示方**：通过健康检查、日志、压测和数据一致性结果证明系统可靠性。

## 本项目负责什么

- 拍品、规则、开拍、取消、落锤、成交订单等核心业务状态机。
- 出价校验、固定加价、0 元起拍、封顶价自动成交、结束前自动延时。
- WebSocket 房间同步：价格、倒计时、排名、领先/被超越/延时/结束提醒。
- MySQL/GORM 持久化核心业务数据，Redis 承担高频读写、幂等、事件流辅助能力。
- 用户账号、JWT、角色权限、主播/场控/买家等基础权限边界。
- Consul 注册发现、健康检查、Docker Compose 本地一键演示。
- 统一 API / WebSocket 契约，供 PC 前端和 H5 端复用。

## 本项目不负责什么

- 不实现主播团队 PC 页面视觉和交互，那是 `live-auction-bid-frontend` 的职责。
- 不实现观众移动端页面，那是 `live-auction-user-h5` 的职责。
- 不为了演示保留 mock/fallback 数据路径；后端主链路必须是真实业务实现。

## 和其它项目的关系

- `live-auction-bid-frontend`：主播团队 PC 工作台，调用本项目 API/WebSocket。
- `live-auction-user-h5`：买家/观众移动 H5，调用本项目 API/WebSocket。
- 根级全栈/归档项目：只用于历史参考或总体说明，当前后端主实现以本目录为准。
